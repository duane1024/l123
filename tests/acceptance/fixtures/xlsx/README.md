# XLSX acceptance fixtures

Binary `.xlsx` files that acceptance transcripts load via `/File Retrieve`
to exercise xlsx-specific features (alignment, borders, merges, …) that
L123 has no user-facing command to set.

## How these were built

Each fixture is generated by `cargo run -p l123-engine --example
build_fixtures`. See `crates/l123-engine/examples/build_fixtures.rs` —
the script is the source of truth; the committed binaries are treated
as generated artifacts so `cargo test` works without re-running it.

This matches option (b) from `docs/XLSX_IMPORT_PLAN.md` §1: the pure-Rust
path. A symmetric read + write bug in IronCalc will slip past these
fixtures; for coverage of that class of bug we would need fixtures
authored by Excel itself (option c), imported here with provenance
documented per-file.

## Updating a fixture

1. Edit the corresponding `build_<feature>` fn in
   `crates/l123-engine/examples/build_fixtures.rs`.
2. `cargo run -p l123-engine --example build_fixtures`.
3. Commit the regenerated `.xlsx` alongside the code change.

## Current fixtures

| File | Feature | Shape |
|---|---|---|
| `alignment.xlsx` | Per-cell horizontal + vertical + wrap alignment (§2.1) | A1=left, B1=center, C1=right, D1=left-aligned number, E1=top+wrap |
| `fill.xlsx` | Per-cell solid background color (§2.3) | A1=#FF0000, B1=#00C800, C1=#3366CC, D1=no fill |
| `sheet_color.xlsx` | Per-sheet tab color (§2.6) | 3 sheets: Overview=no color, Q1 Red=#DBBE29, Q2 Blue=#3366CC. Built by hand-patching IronCalc's output (its exporter drops tabColor). |
| `font.xlsx` | Per-cell font fg color + strikethrough (§3.1) | A1=#FF0000, B1=#3366CC, C1=struck, D1=plain |
| `borders.xlsx` | Per-cell right-side border styles (§2.2) | A1=Thin, B1=Thick, C1=Double, D1=Dashed, E1=plain. Dotted omitted — IronCalc importer drops it. |
| `comments.xlsx` | Cell comments (§2.7) | A1='first note', B2='second note', C3=plain. IronCalc's exporter doesn't emit comments, so the comments part + sheet rels are hand-injected. Authors are dropped on read. |
