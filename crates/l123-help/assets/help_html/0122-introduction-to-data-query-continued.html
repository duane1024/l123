Introduction to /Data Query (continued)

If you want to use more than one database table for the input range,
note the following:

    Name each table with /Range Name Create. Naming the tables makes
   them easier to refer to in the criteria and output ranges.

    When a field name that you want to use in the criteria and/or output
   ranges appears in more than one database table, you must enter the
   table name, then a . (period), and then the field name. For example,
   SALES92.QTR1 refers to QTR1 (the field) in the table named SALES92, and
   SALES93.QTR1 refers to QTR1 (the field) in the table named SALES93.

NOTE  If the current argument separator is a . (period), use a , (comma)
to separate the table and field names.
