Add-ins nested too deeply

Add-ins nested too deeply --  You used a macro in your add-in that invokes
another add-in, that itself invokes another add-in, and so on. The total
number of such nested invocations cannot exceed nine. If you are a developer,
rewrite your add-in so that it does not nest add-ins more than nine layers
deep. Otherwise, you must call your add-in developer.

Cannot find "file_name" --  l123 cannot find the add-in file (a file with a
.PLC or .DLD extension). Make sure you typed the file name correctly and
that l123 can find the file. l123 looks for add-in files in the path you
specify. Otherwise, l123 looks first in the default add-in directory (if
you set one up), then in the current directory, then in the l123 directory.

Cannot find library file "library_file_name" -- You did not copy a library
file included in your application into the l123 directory. If you are a
developer and "library_file_name" is a library you wrote, the library file
may have an invalid header format and cannot be identified; you must correct
the header format. If you are a user and you copied the file, or the file was
not included with the add-in, you should contact your add-in developer.
