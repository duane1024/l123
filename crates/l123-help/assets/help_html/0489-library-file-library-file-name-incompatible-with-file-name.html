Library file "library_file_name" incompatible with "file_name" -- The
version of your add-in does not match the version of the library it uses.
Contact your add-in developer. If you are a developer, you must recompile
your add-in so that the timestamp recorded in the add-in's import table
matches the timestamp of the imported library file. If "library_file_name"
is a library you wrote, you may have modified and recompiled your add-in
primitive file, but not your library file (a file with a .DLL or .DLD
extension). In this case, you must recompile your library file. Make sure
add-in users place your recompiled library file in their l123 program
directory.

No add-in application assigned -- You pressed ALT-F7 (APP1), ALT-F8 (APP2),
or ALT-F9 (APP3) to invoke an application, but no application was assigned
to the key. Select Load or Settings System from the ALT-F10 (ADDIN) menu to
assign the application to a key.
