Debugging a Macro

Debugging a Macro -- If a macro does not produce the results you expected,
you need to edit the macro instructions. Review the macro instructions
carefully to detect mistakes in them, or use STEP mode to debug them.

Common mistakes made when entering macros include the following:
      Spelling errors in commands and range names.
      Missing tildes and missing steps in a command sequence.
      Brackets or parentheses used instead of braces around a key
     name or macro command.
      Incorrect cell and range references.
      Range names and addresses without file references when the cell or
     range is in another file.
      Range names that duplicate macro key names or macro commands.
      Incorrect spaces, missing or incorrect arguments, and misplaced
     argument separators in macro commands and @functions.

If 1-2-3 displays an error message when you run a macro, press F1 (HELP)
while the message is on the screen to see an explanation of the error.
