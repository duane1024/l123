Cannot create names from a range of labels that is not in memory

You were using /Range Name Labels and one of the following problems
occurred:

* You specified a range in a file on disk as the labels range. The
   labels range must be in an active file. Before you use /Range Name
   Labels, use /File Retrieve or Open to read into memory the file that
   contains the labels range.

* When specifying a labels range in another active file, you used a file
   reference that did not include a path and/or extension. Reselect /Range
   Name Labels Right, Down, Left, or Up; press F3 (NAME); and select the
   appropriate file name from the displayed list. 1-2-3 automatically
   includes the path and extension when you use this method to create
   the file reference.
