Error writing CNF file

You selected /Worksheet Global Default Update to update the l123
configuration file, or you pressed ALT-F10 (ADDIN) and selected Settings
System Update to update the add-in configuration file, and l123 could
not complete the command for one of these reasons:

* The disk that contains the configuration file is write-protected. Disable
   write-protection, if possible, and repeat the command.
* The disk that contains the configuration file is full. You need to
   delete one or more files from that disk before you can update the
   CNF file.
* l123 could not access the configuration file because you are already
   using the maximum number of files DOS allows. To solve this problem,
   use /Worksheet Delete File to delete one or more of your active files;
   use /File Admin Reservation Release to release the reservation of
   one or more active files; or end another application program, if one
   is running.
